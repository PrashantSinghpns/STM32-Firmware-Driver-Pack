# Example 06: ADC DMA Read

This example demonstrates how to read ADC values using DMA.

The example uses the ADC driver APIs from:

```text
User_Drivers/
├── adc_driver.c
└── adc_driver.h
```

---

## Objective

Read analog values continuously using ADC with DMA.

This example verifies:

* ADC initialization
* DMA configuration
* ADC buffer reading
* Continuous analog data acquisition
* Reduced CPU involvement compared to polling

---

## Hardware Requirement

* STM32 development board
* Potentiometer or analog sensor
* Jumper wires
* USB cable

---

## ADC + DMA Concept

In polling mode, the CPU waits for ADC conversion to complete.

In DMA mode, ADC conversion results are automatically transferred into a memory buffer.

This is useful for:

* Continuous sensor reading
* Multi-channel ADC sampling
* Battery monitoring
* Audio/signal sampling
* Data acquisition systems

---

## Example Buffer

```c
#define ADC_BUFFER_SIZE  10

uint32_t adc_buffer[ADC_BUFFER_SIZE];
```

The ADC values will be stored inside `adc_buffer`.

---

## main.c Example

```c
#include "stm32f4xx_hal.h"
#include "adc_driver.h"

/*
 * ADC handle.
 * This is usually generated by STM32CubeIDE.
 */
extern ADC_HandleTypeDef hadc1;

/*
 * ADC DMA buffer size.
 */
#define ADC_BUFFER_SIZE  10

/*
 * ADC DMA buffer.
 */
uint32_t adc_buffer[ADC_BUFFER_SIZE];

/*
 * Function prototypes.
 * These functions are usually generated by STM32CubeIDE.
 */
void SystemClock_Config(void);
void MX_GPIO_Init(void);
void MX_DMA_Init(void);
void MX_ADC1_Init(void);

int main(void)
{
    HAL_Init();

    SystemClock_Config();

    MX_GPIO_Init();
    MX_DMA_Init();
    MX_ADC1_Init();

    /*
     * Start ADC in DMA mode.
     * ADC values will be stored automatically in adc_buffer.
     */
    HAL_ADC_Start_DMA(&hadc1,
                      adc_buffer,
                      ADC_BUFFER_SIZE);

    while (1)
    {
        /*
         * Check adc_buffer values using debugger watch window.
         * DMA keeps updating the buffer automatically.
         */
        HAL_Delay(500);
    }
}
```

---

## Optional Callback

This callback is called when the ADC DMA buffer is completely filled.

```c
void HAL_ADC_ConvCpltCallback(ADC_HandleTypeDef *hadc)
{
    if (hadc->Instance == ADC1)
    {
        /*
         * ADC DMA conversion complete.
         * Process adc_buffer here if needed.
         */
    }
}
```

## STM32CubeIDE Configuration

Enable:

```text
ADC1
DMA for ADC1
DMA circular mode
ADC channel, for example PA0
```

Recommended settings:

| Setting        | Value                                         |
| -------------- | --------------------------------------------- |
| ADC Resolution | 12-bit                                        |
| DMA Mode       | Circular                                      |
| Data Width     | Word or Half-word, depending on configuration |
| ADC Mode       | Continuous Conversion                         |
| Buffer         | adc_buffer                                    |

## Troubleshooting

### Buffer values are not changing

Check:

* ADC channel is configured correctly
* DMA is enabled for ADC
* `MX_DMA_Init()` is called before `MX_ADC1_Init()`
* ADC input pin is in analog mode
* Potentiometer/sensor is connected properly

---

### DMA error occurs

Check:

* DMA stream/channel selection
* ADC and DMA linking in CubeIDE
* Buffer data type
* DMA data width settings

---

## Summary
