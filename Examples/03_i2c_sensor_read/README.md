# Example 03: I2C Sensor Read

This example demonstrates how to read a register from an I2C-based sensor using the custom I2C driver.

The example uses the I2C driver APIs from:

```text
User_Drivers/
├── i2c_driver.c
└── i2c_driver.h
```

---

## Objective

Read a sensor register using I2C communication.

This example verifies:

* I2C peripheral initialization
* I2C device addressing
* Register-based sensor reading
* I2C read API
* Basic sensor communication

---

## Hardware Requirement

* STM32 development board
* I2C sensor module
* Jumper wires
* USB cable

Example sensor modules:

* MPU6050
* BMP280
* BME280
* DS3231 RTC
* EEPROM module

---

## I2C Pin Configuration

Common I2C pins depend on the STM32 board and selected I2C peripheral.

Example for many STM32 boards:

| Signal | Description    |
| ------ | -------------- |
| SDA    | I2C data line  |
| SCL    | I2C clock line |
| VCC    | Sensor power   |
| GND    | Ground         |

---

## Example Sensor

This example uses a dummy MPU6050-style register read.

```text
Sensor Address: 0x68
WHO_AM_I Register: 0x75
Expected Value: Usually 0x68 for MPU6050
```

---

## main.c Example

```c
#include "stm32f4xx_hal.h"
#include "i2c_driver.h"

/*
 * I2C handle.
 * This is usually generated by STM32CubeIDE when I2C1 is enabled.
 */
extern I2C_HandleTypeDef hi2c1;

/*
 * Example sensor address and register.
 * For STM32 HAL, 7-bit I2C address is commonly shifted left by 1.
 */
#define SENSOR_ADDRESS       0x68
#define WHO_AM_I_REGISTER    0x75

/*
 * Function prototypes.
 * These functions are usually generated by STM32CubeIDE.
 */
void SystemClock_Config(void);
void MX_GPIO_Init(void);
void MX_I2C1_Init(void);

int main(void)
{
    uint8_t sensor_id = 0;

    /*
     * Initialize HAL library.
     */
    HAL_Init();

    /*
     * Configure system clock.
     */
    SystemClock_Config();

    /*
     * Initialize GPIO and I2C peripherals.
     */
    MX_GPIO_Init();
    MX_I2C1_Init();

    /*
     * Read WHO_AM_I register from sensor.
     */
    I2C_Driver_ReadRegister(&hi2c1,
                            SENSOR_ADDRESS << 1,
                            WHO_AM_I_REGISTER,
                            &sensor_id,
                            1,
                            1000);

    while (1)
    {
        /*
         * In debugging mode, watch the sensor_id variable.
         * For MPU6050, expected value is usually 0x68.
         */
    }
}
```

## Expected Output

There is no LED or UART output in this basic example.

Expected result:

```text
sensor_id = 0x68
```

You can check this value using:

* STM32CubeIDE debugger
* Watch window
* Live expression
* UART print, if UART logging is added

## Test Steps

1. Connect the I2C sensor to STM32.
2. Enable I2C1 in STM32CubeIDE.
3. Configure I2C speed, usually 100 kHz or 400 kHz.
4. Generate code.
5. Add `i2c_driver.c` and `i2c_driver.h`.
6. Include `i2c_driver.h` in `main.c`.
7. Build the project.
8. Flash the firmware.
9. Start debugging.
10. Check the `sensor_id` variable.

## Troubleshooting

### Sensor ID is 0x00 or 0xFF

Check:

* SDA and SCL wiring
* VCC and GND connection
* Correct I2C address
* Address shifted left by 1
* Pull-up resistors on SDA and SCL
* Correct I2C peripheral selected

---

### I2C communication fails

Check:

* Sensor datasheet for correct register address
* I2C clock speed
* Sensor power voltage
* Common ground between STM32 and sensor
* Whether the sensor module already has pull-up resistors

---

### Compilation error: i2c_driver.h not found

Add the `User_Drivers/` folder to the include path in STM32CubeIDE.
