# Example 02: UART Logging

This example demonstrates how to send debug messages from an STM32 board to a serial terminal using the custom UART driver.

The example uses the UART driver APIs from:

```text
User_Drivers/
├── uart_driver.c
└── uart_driver.h
```

---

## Objective

Send debug log messages over UART.

This example verifies:

* UART peripheral initialization
* UART transmit function
* String printing
* Serial terminal output
* Basic firmware logging

---

## Hardware Requirement

* STM32 development board
* USB cable
* Serial terminal software such as PuTTY, Tera Term, RealTerm, or STM32CubeIDE serial terminal

---

## UART Configuration

Common UART settings:

| Setting      | Value  |
| ------------ | ------ |
| Baud Rate    | 115200 |
| Data Bits    | 8      |
| Parity       | None   |
| Stop Bits    | 1      |
| Flow Control | None   |

## main.c Example

```c
#include "stm32f4xx_hal.h"
#include "uart_driver.h"

/*
 * UART handle.
 * This is usually generated by STM32CubeIDE when USART2 is enabled.
 */
extern UART_HandleTypeDef huart2;

/*
 * Function prototypes.
 * These functions are usually generated by STM32CubeIDE.
 */
void SystemClock_Config(void);
void MX_GPIO_Init(void);
void MX_USART2_UART_Init(void);

int main(void)
{
    /*
     * Initialize HAL library.
     */
    HAL_Init();

    /*
     * Configure system clock.
     */
    SystemClock_Config();

    /*
     * Initialize GPIO.
     */
    MX_GPIO_Init();

    /*
     * Initialize USART2.
     */
    MX_USART2_UART_Init();

    /*
     * Print startup message.
     */
    UART_Driver_Print(&huart2, "STM32 UART Logging Started\r\n");

    while (1)
    {
        /*
         * Print message every 1 second.
         */
        UART_Driver_Print(&huart2, "System running...\r\n");

        /*
         * Wait for 1000 milliseconds.
         */
        HAL_Delay(1000);
    }
}
```

## Test Steps

1. Open the STM32 project in STM32CubeIDE.
2. Enable USART2 in asynchronous mode.
3. Set baud rate to 115200.
4. Generate code.
5. Add `uart_driver.c` and `uart_driver.h` to the project.
6. Include `uart_driver.h` in `main.c`.
7. Build the project.
8. Flash the firmware to the board.
9. Open serial terminal.
10. Select correct COM port and baud rate.
11. Check if log messages are printed.

## Troubleshooting

### No output on serial terminal

Check:

* Correct COM port is selected
* Baud rate is set to 115200
* USART2 is enabled
* TX/RX pins are configured correctly
* USB cable supports data transfer
* `uart_driver.c` is added to the build
* Serial terminal is opened after flashing

---

### Garbage characters are displayed

Check:

* Baud rate mismatch
* Wrong UART selected
* Incorrect clock configuration

---

### Compilation error: uart_driver.h not found

Add the `User_Drivers/` folder to the include path in STM32CubeIDE.
