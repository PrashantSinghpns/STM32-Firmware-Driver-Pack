# Example 04: SPI Device Read

This example demonstrates how to communicate with an SPI device using the custom SPI driver.

The example uses the SPI driver APIs from:

```text
User_Drivers/
├── spi_driver.c
└── spi_driver.h
```

---

## Objective

Read data from an SPI device using transmit/receive communication.

This example verifies:

* SPI peripheral initialization
* Manual chip select control
* SPI transmit
* SPI receive
* SPI transmit/receive operation
* Basic SPI device communication

---

## Hardware Requirement

* STM32 development board
* SPI device/module
* Jumper wires
* USB cable

Example SPI devices:

* SPI flash memory
* SPI display
* External ADC
* SPI sensor
* RF module

## SPI Pin Configuration

Common SPI signals:

| Signal | Description                |
| ------ | -------------------------- |
| MOSI   | Master Out Slave In        |
| MISO   | Master In Slave Out        |
| SCK    | Serial Clock               |
| CS/SS  | Chip Select / Slave Select |
| VCC    | Power                      |
| GND    | Ground                     |

---

## Example Command

This example uses a dummy SPI flash-style command:

```text
Command: 0x9F
Purpose: Read device ID / JEDEC ID
```

Many SPI flash devices use `0x9F` as a read identification command.

---

## main.c Example

```c
#include "stm32f4xx_hal.h"
#include "spi_driver.h"

/*
 * SPI handle.
 * This is usually generated by STM32CubeIDE when SPI1 is enabled.
 */
extern SPI_HandleTypeDef hspi1;

/*
 * Chip select pin.
 * Change this according to your board connection.
 */
#define SPI_CS_PORT    GPIOA
#define SPI_CS_PIN     GPIO_PIN_4

/*
 * Function prototypes.
 * These functions are usually generated by STM32CubeIDE.
 */
void SystemClock_Config(void);
void MX_GPIO_Init(void);
void MX_SPI1_Init(void);

int main(void)
{
    /*
     * 0x9F is commonly used as a read-ID command for SPI flash devices.
     */
    uint8_t tx_data[4] = {0x9F, 0x00, 0x00, 0x00};
    uint8_t rx_data[4] = {0};

    /*
     * Initialize HAL library.
     */
    HAL_Init();

    /*
     * Configure system clock.
     */
    SystemClock_Config();

    /*
     * Initialize GPIO and SPI.
     */
    MX_GPIO_Init();
    MX_SPI1_Init();

    /*
     * Select SPI slave device.
     * Chip select is usually active LOW.
     */
    SPI_Driver_CS_Low(SPI_CS_PORT, SPI_CS_PIN);

    /*
     * Send command and receive response.
     */
    SPI_Driver_TransmitReceive(&hspi1,
                               tx_data,
                               rx_data,
                               4,
                               1000);

    /*
     * Deselect SPI slave device.
     */
    SPI_Driver_CS_High(SPI_CS_PORT, SPI_CS_PIN);

    while (1)
    {
        /*
         * In debug mode, watch rx_data array.
         * rx_data may contain device ID bytes depending on the connected SPI device.
         */
    }
}
```

## Test Steps

1. Connect the SPI device to STM32.
2. Enable SPI1 in STM32CubeIDE.
3. Configure SPI mode according to the device datasheet.
4. Configure chip select pin as GPIO output.
5. Generate code.
6. Add `spi_driver.c` and `spi_driver.h`.
7. Include `spi_driver.h` in `main.c`.
8. Build the project.
9. Flash the firmware.
10. Start debugging.
11. Check the `rx_data` buffer.

## Troubleshooting

### No response from SPI device

Check:

* MOSI, MISO, SCK, and CS wiring
* Correct SPI mode
* Correct clock polarity and phase
* Correct chip select pin
* Chip select active LOW behavior
* Device power and ground
* SPI speed is not too high

---

### Response is always 0x00 or 0xFF

Check:

* MISO connection
* Chip select timing
* Correct command byte
* Device datasheet
* SPI peripheral selected correctly

---

### Compilation error: spi_driver.h not found

Add the `User_Drivers/` folder to the include path in STM32CubeIDE.
