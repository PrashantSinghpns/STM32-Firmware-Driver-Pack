# Example 05: ADC Polling Read

This example demonstrates how to read an analog input using the custom ADC driver.

The example uses the ADC driver APIs from:

```text
User_Drivers/
├── adc_driver.c
└── adc_driver.h
```

---

## Objective

Read analog voltage from an ADC channel using polling mode.

This example verifies:

* ADC peripheral initialization
* ADC polling conversion
* Raw ADC value reading
* ADC value to voltage conversion
* Basic analog sensor reading

---

## Hardware Requirement

* STM32 development board
* Potentiometer or analog sensor
* Jumper wires
* USB cable

Example analog input sources:

* Potentiometer
* LDR voltage divider
* Temperature sensor analog output
* Battery voltage divider
* Analog joystick

## ADC Configuration

Common ADC settings:

| Setting           | Value             |
| ----------------- | ----------------- |
| ADC Resolution    | 12-bit            |
| Reference Voltage | 3.3V              |
| Conversion Mode   | Single Conversion |
| Data Alignment    | Right             |
| Method            | Polling           |

For a 12-bit ADC:

```text
Minimum value = 0
Maximum value = 4095
```

## main.c Example

```c
#include "stm32f4xx_hal.h"
#include "adc_driver.h"

/*
 * ADC handle.
 * This is usually generated by STM32CubeIDE when ADC1 is enabled.
 */
extern ADC_HandleTypeDef hadc1;

/*
 * Function prototypes.
 * These functions are usually generated by STM32CubeIDE.
 */
void SystemClock_Config(void);
void MX_GPIO_Init(void);
void MX_ADC1_Init(void);

int main(void)
{
    uint32_t adc_value = 0;
    float voltage = 0.0f;

    /*
     * Initialize HAL library.
     */
    HAL_Init();

    /*
     * Configure system clock.
     */
    SystemClock_Config();

    /*
     * Initialize GPIO and ADC.
     */
    MX_GPIO_Init();
    MX_ADC1_Init();

    while (1)
    {
        /*
         * Read ADC value using polling mode.
         */
        ADC_Driver_ReadPolling(&hadc1,
                               &adc_value,
                               1000);

        /*
         * Convert raw ADC value into voltage.
         * For 12-bit ADC, resolution is 12.
         * Reference voltage is usually 3.3V.
         */
        voltage = ADC_Driver_ConvertToVoltage(adc_value,
                                              3.3f,
                                              12);

        /*
         * Check adc_value and voltage using debugger watch window.
         */
        HAL_Delay(500);
    }
}
```

## Expected Output

There is no UART or LED output in this basic example.

Expected result:

```text
adc_value changes according to analog input voltage
voltage shows the calculated voltage
```

Example values:

| Input Voltage | Approx ADC Value |
| ------------- | ---------------- |
| 0V            | 0                |
| 1.65V         | 2048             |
| 3.3V          | 4095             |

---

## Voltage Conversion Formula

```text
Voltage = (ADC_Value / 4095) × 3.3
```

Example:

```text
ADC_Value = 2048
Voltage = (2048 / 4095) × 3.3
Voltage ≈ 1.65V
```

## Test Steps

1. Connect potentiometer output to ADC input pin.
2. Connect potentiometer side pins to 3.3V and GND.
3. Open STM32CubeIDE.
4. Enable ADC1.
5. Select one ADC input channel, for example PA0.
6. Set ADC resolution to 12-bit.
7. Generate code.
8. Add `adc_driver.c` and `adc_driver.h`.

10. Build and flash the firmware.
11. Start debug mode.

## Troubleshooting

### ADC value is always 0

Check:

* ADC pin is configured in analog mode
* Correct ADC channel is selected
* Sensor/potentiometer is connected properly
* Ground is common between sensor and STM32
* ADC initialization function is called

---

### ADC value is always 4095

Check:

* Input pin may be connected directly to 3.3V
* Sensor output may be saturated
* Wrong ADC channel may be selected

---

### ADC value is unstable

Check:

* Loose jumper wires
* Floating analog input
* Missing ground connection
* Noisy sensor output

You can improve stability later using averaging or filtering.
