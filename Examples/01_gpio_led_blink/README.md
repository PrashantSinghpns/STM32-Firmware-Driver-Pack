# Example 01: GPIO LED Blink

This example demonstrates how to blink an LED using the custom GPIO driver.

The example uses the GPIO driver APIs from:

```text
User_Drivers/
├── gpio_driver.c
└── gpio_driver.h
```

---

## Objective

Blink an LED using GPIO output mode.

This example verifies:

* GPIO clock enable
* GPIO pin initialization
* GPIO output mode
* GPIO pin toggle
* Delay-based LED blinking

---

## Hardware Requirement

* STM32 development board
* On-board LED or external LED
* USB cable for programming and power

For many STM32 Nucleo boards, the user LED is connected to `GPIOA PIN 5`.

If your board uses a different LED pin, update the port and pin in the code.

---

## Pin Configuration

| Component | GPIO Port | GPIO Pin   | Mode   |
| --------- | --------- | ---------- | ------ |
| LED       | GPIOA     | GPIO_PIN_5 | Output |

---

## main.c Example

```c
#include "stm32f4xx_hal.h"
#include "gpio_driver.h"

/*
 * LED port and pin definition.
 * For many STM32 Nucleo boards, the user LED is connected to PA5.
 */
#define LED_PORT    GPIOA
#define LED_PIN     GPIO_PIN_5

/*
 * Function prototype for system clock configuration.
 * This function is usually generated by STM32CubeIDE.
 */
void SystemClock_Config(void);

int main(void)
{
    /*
     * Initialize the HAL library.
     * This sets up Flash interface, SysTick timer, and low-level hardware support.
     */
    HAL_Init();

    /*
     * Configure the system clock.
     * In STM32CubeIDE projects, this function is usually auto-generated.
     */
    SystemClock_Config();

    /*
     * Enable clock for GPIOA port.
     * GPIO peripheral clock must be enabled before using GPIOA pins.
     */
    __HAL_RCC_GPIOA_CLK_ENABLE();

    /*
     * Initialize LED pin as output.
     * No pull-up or pull-down is required for LED output.
     * Low speed is enough for simple LED blinking.
     */
    GPIO_Driver_Init(LED_PORT,
                     LED_PIN,
                     GPIO_DRIVER_MODE_OUTPUT,
                     GPIO_DRIVER_NOPULL,
                     GPIO_DRIVER_SPEED_LOW);

    /*
     * Infinite loop.
     * The LED will toggle every 500 milliseconds.
     */
    while (1)
    {
        /*
         * Toggle LED pin.
         * If LED is ON, it becomes OFF.
         * If LED is OFF, it becomes ON.
         */
        GPIO_Driver_TogglePin(LED_PORT, LED_PIN);

        /*
         * Wait for 500 milliseconds.
         */
        HAL_Delay(500);
    }
}
```

---

## Expected Output

The LED should blink continuously.

Blink rate:

```text
500 ms ON
500 ms OFF
```

---

## Test Steps

1. Open the STM32 project in STM32CubeIDE.
2. Add `gpio_driver.c` and `gpio_driver.h` to the project.
3. Include `gpio_driver.h` in `main.c`.
4. Enable GPIOA clock.
5. Configure PA5 as output.
6. Build the project.
7. Flash the firmware to the STM32 board.
8. Check if the LED blinks.

---

## Troubleshooting

### LED is not blinking

Check:

* Correct GPIO port is selected
* Correct GPIO pin is selected
* GPIO clock is enabled
* `gpio_driver.c` is added to the build
* `gpio_driver.h` include path is configured
* Board is properly powered
* Firmware is flashed successfully

---

### Compilation error: gpio_driver.h not found

Add the `User_Drivers/` folder to the include path in STM32CubeIDE.

---

### Compilation error: undefined reference to GPIO_Driver_TogglePin

Make sure `gpio_driver.c` is included in the project source files.

---

## Summary

This example confirms that the custom GPIO driver can initialize and control a GPIO output pin successfully.
